## Checklist Before Publishing

Copy this checklist for each feature page:

### Content Completeness
- [ ] Headline follows proven formula
- [ ] Subheadline expands on benefit
- [ ] Value proposition in one clear sentence
- [ ] Problem section paints pain vividly
- [ ] 3-5 benefits with before/after/result
- [ ] Technical explanation with code example
- [ ] Social proof (testimonials, stats, or both)
- [ ] 3-5 common objections answered
- [ ] Risk reversal included
- [ ] Related features listed
- [ ] All bracketed placeholders replaced

### Optimization
- [ ] Specific numbers throughout (not vague)
- [ ] "You" language used consistently
- [ ] Active voice (not passive)
- [ ] Short paragraphs (2-3 lines)
- [ ] CTAs above fold and after each section
- [ ] Social proof as early as possible
- [ ] Benefits emphasized over features
- [ ] Outcomes quantified

### Technical
- [ ] Frontmatter filled correctly
- [ ] Status matches template (completed/beta/planned)
- [ ] All required fields present
- [ ] Valid ISO dates
- [ ] Tags relevant and searchable
- [ ] Feature ID matches URL slug
- [ ] Category correct

### Quality
- [ ] Spell checked
- [ ] Grammar checked
- [ ] Links work
- [ ] Code examples test correctly
- [ ] Testimonials are real (with permission)
- [ ] Stats are current and accurate
- [ ] No marketing fluff or hype
- [ ] Reads like helping a friend, not selling

---

## Copy Guidelines Reminder

**Voice:** Authority + warmth + technical depth
**Think:** Wired + Ars Technica + Alex Hormozi
**Not:** Corporate buzzwords or empty hype

**Every sentence should:**
- Teach something OR
- Prove something OR
- Sell something (through teaching/proving)

**If it doesn't do one of those three things, delete it.**
